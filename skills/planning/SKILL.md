# Project Planning

## When to Use
- Starting a new feature or project
- Breaking down complex tasks
- Creating roadmaps
- Estimating effort
- Organizing MVP scope

## Planning Framework

### 1. Clarify the Goal
- What problem are we solving?
- Who is this for?
- What does success look like?

### 2. Identify Key Milestones
- Break into 2-4 major phases
- Each phase should deliver value
- Define dependencies between phases

### 3. Task Breakdown
- Break each milestone into specific tasks
- Distinguish between:
  - **Must have** (MVP essentials)
  - **Should have** (important but not critical)
  - **Nice to have** (can ship later)

### 4. Estimate Effort
- Use relative sizing (T-shirt: S/M/L/XL)
- Add buffer for unknowns (20-30%)
- Identify where you need help

### 5. Define Next Steps
- What can be done immediately?
- What's blocked on research?
- What's blocked on decisions?

## Output Format
```markdown
## Goal: [One sentence]

## Phases
### Phase 1: [Name]
- [ ] Task 1
- [ ] Task 2

### Phase 2: [Name]
- [ ] Task 3

## Risks
- [Risk 1 and mitigation]

## Next Steps
1. [Immediate action]
2. [Second immediate action]
```

## Pro Tips
- Start small - ship v1 fast
- Validate before building
- Reduce scope, not quality
- Document decisions
